# 00. Course Gateway: Learn Like a Beginner, Build Like an Engineer

## Core idea

This course is arranged so you can learn a concept in this order:

1. Read it like a story.
2. See the picture mental model.
3. Try one tiny exercise.
4. Explain the idea in one short sentence.

If you complete these 4 steps once, you have really started learning.

## The course map

```
[Understand] -> [Visualize] -> [Practice] -> [Explain] -> [Review]
```

- [Understand] = read one topic in small chunks.
- [Visualize] = use the diagrams and arrays/boxes inside pages.
- [Practice] = solve 1-2 simple problems in your chosen language.
- [Explain] = say what changed in 1 short line.
- [Review] = revisit the topic after 2 days and fix one mistake.

## Learning levels

- `Starter`: arrays, strings, pointers, basics.
- `Builder`: recursion, trees, hash, intervals, heaps.
- `Mastery`: dynamic programming, graphs, system-like hard mixes.

## 14-day jumpstart (first plan)

1. Read one topic per day from `docs/01-foundation`.
2. Solve 2 tiny problems using the same pattern.
3. Record 1 mistake in your notes.
4. Revisit the same topic on day 4 and day 7.

## Two learning modes

- `Normal mode`: move faster, read code-heavy sections first.
- `Kid-friendly mode`: read every heading, draw the shape, then code.

Kid-friendly mode checklist:

- Replace every new word with a toy/real-world object.
- Draw only one page before opening the code.
- Ask yourself: "What happens if I move this one step?"

## What to do if a topic feels hard

- Stop and shrink the example to 1 line.
- Draw 3 input/output arrows with pencil.
- Implement only the smallest function first.
- Then add full logic after your explanation passes the "does this make sense?" test.
