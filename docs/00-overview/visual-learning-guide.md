# Visual Learning Guide

## How this course teaches

Every lesson uses this exact flow:

1. Story: understand the concept with simple object analogy.
2. Shape: a tiny visual map or sequence.
3. Action: change one pointer, one index, one variable.
4. Repeat: solve 1 tiny problem.

## Visual map templates

### 1) Step-by-step movement

Use this when you see loops, two pointers, or window slides.

```
start  ->  check -> move one side -> check again -> final answer
```

### 2) Node connection map

Use this for linked structure topics.

```
value_1 -> value_2 -> value_3 -> None
```

### 3) State staircase

Use this for DP topics.

```
state(0) -> state(1) -> state(2) -> ... -> state(n)
```

## 5-minute study routine

- Minute 1: read heading and bullet points.
- Minute 2: draw the template above.
- Minute 3: write pseudo-code in 3-5 lines.
- Minute 4: code one tiny version.
- Minute 5: explain one sentence aloud.

## Safe learning guardrails

- Never skip the visual step.
- Never optimize before correctness.
- Never start a new topic before closing the previous 5-minute review.
- If anything is unclear, convert it into a toy story.

## Beginner phrasebook

- `state`: what is remembered right now.
- `transition`: how we move to the next state.
- `window`: current active group of items.
- `pointer`: the current hand that points to something.
- `next`: the bridge from current to later item.
